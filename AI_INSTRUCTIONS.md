## ⚠️ 絶対に守るべきルール

### コード管理の鉄則
1. **既存のコードは絶対に削除しない**
2. **動いているコードは絶対に書き換えない**
3. **新機能は既存コードの上に追加する**
4. **「簡略化」「リファクタリング」「スタブ化」を勝手にやらない**

### 作業の進め方
1. 実装前に「何を作るか」を明確に説明
2. 承認を得てから実装開始
3. 実装後、動作確認方法を説明
4. エラーが出たら即座に報告し、自分で修正

### 絶対にやってはいけないこと
- ユーザーに「このコマンドを実行してください」と丸投げ
- 「簡略化します」「リファクタリングします」を勝手に実行
- 動いているコードを「改善」のため書き換え
- 新しいファイルを追加したのに古いファイルを削除しない（結果的に壊れる）
- 問題が起きたときに言い訳や説明ばかりする
- 「元に戻せ」= ブラウザで起動してプレイできる状態に戻すこと（コードだけ戻すのではない）

## 🔧 技術方針

### プラットフォーム
- **Web（HTML5）のまま**
- 理由：
  - ブラウザ上でエディタを動かすのが最適
  - ユーザーがアクセスしやすい
  - データ共有が簡単
  - 必要ならElectronでexe化も可能

### 技術スタック
- Pure JavaScript（フレームワークなし）
- HTML5 Canvas
- タイルベースマップシステム
- プロシージャルテクスチャ生成（Phase 2完成までの暫定）
- プラグインベースアーキテクチャ

### テクスチャについて
- 当面は「仮のテクスチャ」で進める
- Phase 2のエディタ完成後、ユーザーが自分で作れるようになる
- AIが勝手にセンスのないテクスチャを作るのは避ける
- 具体的な指示があった場合のみ作成する

### パフォーマンス改善
```javascript
// 既に実装済み
✓ バッチ描画システム
✓ オフスクリーン描画の最適化

// 今後追加するもの
□ Web Worker（重い処理を分散）
□ WebGL移行（必要になった場合）
□ メモリ管理の最適化
```
